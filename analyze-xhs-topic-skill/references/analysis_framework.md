# 小红书选题分析框架

本文档定义了小红书选题分析的标准化方法和输出格式。

## 分析目标

通过分析热门笔记，提取可复用的方法论，为账号人设生成定制选题。

## 分析维度

### 1. 核心理念分析

**目标**：1-2句话总结热门笔记主要传递什么信息

**分析要点**：
- 主要观点是什么？
- 传递了什么价值观？
- 情绪倾向：积极/消极/中立？

**输出格式**：
```
主要观点：[1-2句话]
情绪倾向：[积极/消极/中立]
```

### 2. 火爆原因分析（Top 3）

**目标**：分析笔记数据火爆的3个关键原因

**分析维度**：

**情绪价值**：
- 给读者什么情绪？
- 确认感、共鸣、好奇、愤怒、希望？

**社会价值**：
- 是否涉及热门话题/社会现象？
- 是否戳中时代痛点？

**实用价值**：
- 是否提供方法/技巧？
- 是否有可复制性？

**输出格式**：
```
1. [情绪价值] 具体说明
2. [社会价值] 具体说明
3. [实用价值] 具体说明
```

### 3. 选题特征分析

**目标**：从选题维度分析热门笔记的特征，找出可复用的选题模式

#### 3.1 目标人群

**分析要点**：
- 这些笔记是写给谁看的？
- 年龄、身份、状态？
- 有什么共同特征？

**输出格式**：
```
主要人群：[人群描述]
- 年龄段：[例如：22-28岁]
- 职业状态：[例如：刚入职、工作3-5年、想裸辞的]
- 核心标签：[例如：迷茫、焦虑、想改变]
```

#### 3.2 痛点与情绪钩子

**分析要点**：
- 这些笔记解决了什么心理需求？戳中了什么痛点？
- 用什么情绪吸引注意？触发什么情绪反应？

**常见痛点类型**：
- 职业迷茫：不知道自己适合什么
- 现状不满：对工作/生活不满意但不知道怎么办
- 焦虑不安：担心未来、担心做错选择
- 认可缺失：需要确认感、需要被理解
- 成长渴望：想要变好、想要突破

**常见情绪类型**：
- 焦虑：担心、害怕、不安
- 渴望：想要、向往、期待
- 愤怒：不满、不公、失望
- 确认：认同、共鸣、被理解
- 好奇：想知道、想了解
- 希望：鼓励、正能量、可能性

**输出格式**：
```
核心痛点 + 情绪钩子：
1. [痛点类型] → [情绪类型]：具体说明如何触发
2. [痛点类型] → [情绪类型]：具体说明如何触发
3. [痛点类型] → [情绪类型]：具体说明如何触发
```

#### 3.3 内容形式与切入角度

**分析要点**：
- 图文还是视频？封面风格、内容长度？
- 从什么角度切入话题？

**形式类型**：
- 图文/视频比例
- 封面风格：大字报、真人+文字、拼图等
- 内容长度：短篇（300字内）、中篇（500-800字）、长篇（800字+）

**切入角度**：
- 个人经历：我的故事、我的选择
- 反常识：颠覆认知、与大众看法相反
- 具体场景：某个特定时刻/场景的描述
- 数据/案例：用数据或案例说话
- 对比反差：前后对比、正反对比

**输出格式**：
```
内容形式：
- 主要形式：[图文/视频比例]
- 封面风格：[风格描述]
- 内容长度：[长度描述]

切入角度：
1. [角度类型]：占比约X%，说明
2. [角度类型]：占比约X%，说明
```

#### 3.4 价值点与冲突反差

**分析要点**：
- 提供了什么别人不知道的信息？给了什么价值？
- 制造了什么冲突或反差？用什么对比吸引注意？

**价值类型**：
- 认知升级：打破认知盲区
- 方法技巧：可操作的具体方法
- 经验分享：真实经历和教训
- 资源推荐：工具、书籍、课程
- 行业真相：内幕、真相、潜规则

**冲突类型**：
- 预期vs现实：理想和现实的差距
- 大众vs个人：和普遍认知相反的选择
- 前后对比：变化前后对比
- 正反对比：两种选择/观点对比
- 数字反差：年龄、收入、时间的对比

**输出格式**：
```
核心价值：
- [价值类型]：具体说明
- [价值类型]：具体说明

冲突设计：
1. [冲突类型]：示例说明
2. [冲突类型]：示例说明
```

#### 3.5 标题公式

**目标**：总结常见的标题模式

**分析方法**：
- 收集所有标题
- 归纳共同模式
- 提取公式结构

**常见标题类型**：
1. **数字+结果+情绪**
   - 示例："26岁刚入职就裸辞，但我不后悔"
   - 公式："[年龄/时间]+[动作]+[转折]+[情绪/态度]"

2. **反常识+悬念**
   - 示例："年薪50w却裸辞，去过另一种生活"
   - 公式："[反差]+[动作]+[悬念]"

3. **具体场景+共鸣**
   - 示例："入职即离职，及时止损"
   - 公式："[具体场景]+[行动/态度]"

4. **问句+痛点**
   - 示例："裸辞前担心的事情发生了么"
   - 公式："[问题]+[痛点场景]"

**输出格式**：
```
1. [公式名称]
   公式：[公式结构]
   示例：[1-2个实际标题]

2. [公式名称]
   ...
```

### 4. 定制选题生成

**目标**：基于用户账号人设生成定制选题

**用户账号人设**（从 `/Users/it/Documents/AI/Claude Code/user.md` 读取）：
- 账号定位：产品经理分享产品设计、产品方法、职场经验
- 内容方向：产品设计、内容创作、工作方法、内容运营
- 写作风格：专业但自然、真实经历、克制但有力度

**选题分类**：

1. **个人经历类**
   - 基于用户真实经历
   - 展现成长和反思
   - 示例："裸辞8个月的3个关键决定"

2. **方法分享类**
   - 可复制的方法论
   - 实用技巧
   - 示例："产品经理如何用杠杆思维提升绩效"

3. **观点输出类**
   - 颠覆认知的观点
   - 职场真相
   - 示例："绩效不是奖励机制，而是存量博弈"

**选题要求**：
- 每个选题 10-50 字
- 数量灵活：通常 8-15 个
- 必须符合账号人设
- 具有可操作性

**输出格式**：
```
### 个人经历类
1. [选题]（预计字数）

### 方法分享类
1. [选题]（预计字数）

### 观点输出类
1. [选题]（预计字数）
```

## 数据处理规范

### 笔记类型区分

**图文笔记**：
- 提取：标题 + 正文
- 数据字段：`desc`

**视频笔记**：
- 提取：标题 + 描述
- 数据字段：`desc`
- 注意：不提取视频脚本

### 数据清洗

- 删除重复内容
- 去除广告信息
- 统一格式
