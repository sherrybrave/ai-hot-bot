# 如果你想明年晋升，现在就得做以下准备了

**分类**：problem-solution-list
**主题**：晋升、职场规划
**风格**：干货类
**来源**：小红书 @夏树
**添加时间**：2026-01-30

---

最近又到了各大厂的晋升季，很多同学还在等消息，或者知道这次没机会，而被提报的同学已经在偷偷准备答辩材料了

我去年升到3-1（管理岗），回头看最大的感受：晋升不是提报时决定的，而是提前一年埋下结果了
如果你有明年晋升的想法，现在就得做以下准备：

1、提前1年和leader提晋升诉求
功夫在平日，早起的鸟儿有虫吃。早早和+1多提自己的晋升诉求，获取对方支持和建议。我就是太晚提，整整晚了半年以上

至少提前半年甚至1年和leader沟通什么时候提报晋升，晋升成功标准是什么样，怎么准备：比如要到下一职级的话，接下来要做哪几个方向，拿到什么成果，才有较大概率晋升。如果到提报时间点才临时准备，肯定来不及啦

2、拿到下一职级的能力模型
最重要的来了，只有清楚当前职级到下一职级的能力模型有什么进阶标准，才能有的放矢做好准备。

以P7-P8产品岗的能力模型为例，4点变化：
（1）从IC（个人贡献者）成长为leader（管理者）

👊不用等晋升后才开始，现在就能做：
虚线带新人/实习生，帮忙梳理思路、制定计划，让ta负责部分模块，你做兜底和指导；
留出更多精力和时间做规划，而不是执行细节

（2）负责1个或多个产品模块 -> 负责1个或多个复杂的业务方向，scope更大（从具体产品模块变成更大更虚的业务方向）

（3）从产品设计 -> 产品规划，不仅有具体方案的输出和落地能力，还得对复杂业务方向有判断力、规划力以及行业洞察力

（4）强调非职权影响力，团队内能管理好成员，对外协同好合作和资源，比如作为owner统筹项目、和比自己级别高的其他团队ld有良好合作等

3、对照能力要求，有意识地选项目、拿结果
不是所有项目都对晋升有用，优先选：
业务价值高：能写进+1、+2甚至+3的OKR；
0-1开拓性项目：比如新业务方向、新模块，能体现你的判断力和开拓能力；
跨部门统筹项目：比如协调多个部门的专项

如果你现在突然意识到："好像我还没什么准备。"

恭喜你，刚刚好，栽下一棵树最好的时间是现在

除了这些，晋升还有很多门道，比如文档怎么写有亮点？评委有哪些必问雷区？我都会在接下来几周更新。朋友们可以蹲蹲关注，稳稳晋升
