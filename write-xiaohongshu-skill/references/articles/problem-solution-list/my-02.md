# 又到OKR月，大厂新人的第一关能有多难

**分类**：problem-solution-list
**主题**：职场经验、OKR
**风格**：干货类
**来源**：小红书 @夏树
**添加时间**：2026-01-30

---

一般大厂的OKR都是季度制，每年3、6、9、12月就是OKR月，要完成上季度OKR总结和规划，这也是除了绩效期外最难的时期。每到这个时期，公司上下长达一个月时间全力搞OKR，几乎都要扒掉一层皮。我们来聊聊为啥现在的大厂OKR这么难？即将入职或者刚来的新人可以多做些准备，和同事领导沟通学习，可别被吓跑了。

首先，OKR总结要求高。不仅要写目标完成度，还要做严格的归因，看业务动作带来了多少贡献，甚至会在每周追动目标完成情况。此外，事项总结也不只是要写做了什么事、是什么进展、回收了多少实验收益，还要写认知learning、后续计划。为了确保自己能在OKR总结时有足够的收益，大家也会早早把要回收实验的需求push起来。

再次，OKR规划要求高。对于下季度规划，每次都要有新故事，请注意是区别于上季度的新故事线，而不是在原有规划上的延续。写规划也不是一两天的事情，大概要比内部对OKR的时间，再提前半个月启动。内部对齐之后，还要赶在和+2、+3对OKR前先对一轮规划，让老板们认可新故事，写进OKR才比较稳妥。

最后，沟通协作也很难。众所周知的原因，大厂人多又聪明，跨团队协作非常难，具体可以看我上一篇文章。往往在OKR内部对完后，就要紧锣密鼓的和所有依赖方约会、沟通OKR，以达成自己的事情能写到对方OKR里去的目的，如果没做这件事，大概率你的OKR后面会得不到支持很难推进。以我们团队来举例，作为产品团队，我们要和同一业务的职能包括研发、设计、算法、运营等沟通，还要和不同业务的产品、研发、算法等沟通，经常要沟通十几个合作方，去另外一个地方出差，从早到晚连轴开会，很多事情还要沟通好几次才能搞定。

总的来说，大厂OKR对总结、规划要求很高，沟通协作也带来极大的工作难度，不仅要搞业务还要搞人际关系，工作量直接double。
