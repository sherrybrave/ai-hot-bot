# 多线程把事做好的关键不是时间管理

**分类**：theory-application
**主题**：多线程、高度协同
**风格**：深度分析类
**来源**：别人
**添加时间**：2026-01-30

---

是高度协同

如图片，今年主包一直在同期做多线程的事情，但都拿到了意想不到的结果：实习超预期完成，得到了mentor和leader的认可和好评；一边实习一边秋招，8月在很多人还没有开始投递的时候就拿到两个BAT秋招offer；账号也在两个多月，做到了涨粉1w+

我在学校同样一直八爪鱼做事，小组作业、实习、比赛、学生工作、大创，但结果很糟糕，我也不开心

我的时间精力都没变，区别只在于现在我做的所有事都是互相反哺的

秋招面试官的问题让我反思实习中的不足，给我的账号提供新选题；实习工作做的越好，越能助力我在秋招中接招；做账号本质是输出，锻炼的表达、逻辑思维能力，也能复用到我的实习和秋招中；而拍摄虽然占据时间，但是恰好作为工作之余的放松，带给我大大的情绪价值和反馈

我做的所有事都变成了一个飞轮，只要推任何一环，其他环都会跟着转动，并反过来给我助力

说实话，人类大脑并不擅长真正的multitasking [扶额R] 因为多线程的本质就是脑子在多个任务之间进行高速切换，频繁切换的结果只能是每个任务都做得更慢、更差，并且极度疲惫

为什么我说多线程能做好的关键不是时间管理，而是高度协同

因为假如我把我原先做的其中一件事换成一件毫无关联的事，"实习 账号 秋招"换成"实习 考公 秋招 "or "实习 账号 考研"，即使我把原来那套时间管理方法原封不动地用到这套新组合上，结果只会是：脑子在不同知识体系里面疯狂切换进而能量耗尽，几件事彼此之间毫无助力，靠意志力硬撑的各种计划迅速崩溃

所以结论很清晰

几件同时在做的事情的底层逻辑高度协同，都在服务于本科直接就业这个目的，这才是我能拿到结果并且"乐在其中"的根本原因

时间管理能让飞轮转得更快，但不能凭空创造飞轮。再往下推一层，行为和目的保持一致才能事半功倍，单纯追求做事数量叠加、盲目地同时开启很多线程只能一事无成

在开启任何多线程之前 必须先思考，先想清楚"为什么"和"做什么"层面，"怎么做"层面的多线程才会变得高效和可持续；先问自己这些事情是不是服务于一个目的，我手头做的能不能帮助最想干的目标，如果答案是否定的，那么强行多线程只能是虐待自己

不急着学怎么同时做五六件事，先想清楚最终到底想要哪一件事做成。一旦想明白了就会发现，需要做的其实始终就是那一件事，只是用了不同的方式在做
