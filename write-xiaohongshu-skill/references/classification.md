# 分类规则文档

本文档说明如何对参考文章进行分类，确保文章库的组织清晰、便于查找。

## 分类原则

**核心原则**：按**框架差异**和**语料差异**进行分类

- **框架差异**：文章结构、逻辑链条、段落功能的不同
- **语料差异**：表达方式、语气风格、内容类型的不同

## 框架类型定义

### problem-solution-list：问题-解决-列举型

**判断标准**：
- 开头提出问题或痛点
- 主体用数字编号列举解决方案/要点/步骤
- 每个要点有小标题 + 详细说明
- 结尾有总结或互动引导

**典型特征**：
- 开头：现象描述/痛点引入 + 背景铺垫
- 主体：3-4个要点并列，用"1、2、3"或"首先、再次、最后"过渡
- 要点展开：小标题 + 解释 + 例子/方法
- 结尾：总结/互动/引导关注

**适用主题**：
- 干货分享类（方法、技巧、经验）
- 方法论类（SOP、步骤、流程）
- 经验总结类（复盘、反思、收获）

**示例文章**：my-01, my-02, my-03, my-04, my-05, my-06, my-07, my-08, my-09

---

### theory-application：理论-应用型

**判断标准**：
- 开头提出问题或痛点，用个人经历引起共鸣
- 引入权威理论或模型，用例子解释
- 将理论拆解成具体步骤/方法
- 每个步骤有详细说明和示例
- 结尾有鼓励和行动呼吁

**典型特征**：
- 开头：问题/痛点 + 个人经历共鸣
- 理论引入：引用权威理论（神经科学、心理学等），用生动例子解释
- 应用拆解：7个步骤左右，每步都有具体说明和示例
- 结尾：鼓励话语 + 行动呼吁

**适用主题**：
- 方法论类（思维模型、框架）
- 技能提升类（表达、写作、沟通）
- 认知升级类（思维转变、新视角）

**示例文章**：my-10

### rant-list：吐槽列举型

**判断标准**：
- 标题包含"十大"、"至暗时刻"、"那些XX们"等
- 开头简短引入或直接进入列举
- 主体用数字编号（1-10）或 Top10（倒序）列举槽点
- 每个槽点是具体场景描述
- 不提供解决方案，只列举问题
- 结尾有总结、反转或强烈互动

**典型特征**：
- 开头：简短引入或直接进入"1、"
- 主体：10个槽点场景，具体时间、人物、事件
- 情绪递进：槽点越来越严重，情绪越来越强
- 表达直接：不藏着掖着，直白吐槽
- emoji丰富：大量情绪化emoji
- 真实性：常说"全部根据真实事件而写"

**适用主题**：
- 吐槽发泄类（工作槽点、职场现象）
- 情绪共鸣类（让读者感同身受）
- 讽刺幽默类（用幽默表达不满）

**示例文章**：my-11, my-12, my-13, my-14, my-15, my-16, my-17, my-18

### personal-narrative：个人叙事型

**判断标准**：
- 标题包含时间节点（写在25年底）+ 身份标签 + 核心事件
- 开头介绍背景（出生、成长环境）
- 主体按时间线叙事（学业→工作→转折）
- 包含自我剖析和反思
- 结尾有未来规划和互动引导

**典型特征**：
- 开头：背景介绍，成长环境
- 成长线：顺叙发展历程（高光时刻）
- 转折点："但只有我自己知道..."（反差）
- 自我剖析：性格特质、内在冲突
- 行动决定：重大转折（裸辞、转型）
- 未来规划：具体的3件事
- 结尾互动："这是我的故事...你呢？"

**适用主题**：
- 个人成长类（人生转折、重大决定）
- 反思感悟类（年度总结、人生思考）
- 经验分享类（gap、转型、探索）

**示例文章**：my-19

---

### game-metaphor：游戏比喻型

**判断标准**：
- 标题用游戏比喻职场现象
- 开头引入现象 + 核心比喻
- 主体用emoji作为小标题（游戏环节）
- 有比喻对应关系（游戏元素=职场元素）
- 结尾有专业分析和人设强化

**典型特征**：
- 开头：引入现象 + 核心比喻
- emoji分节：🔖🙋🙈🙅‍♀️ 等游戏环节
- 比喻对应：狼人杀=绩效考核，发牌=分活，村民=牛马
- 有叙有议：场景描述 + 专业分析
- 专业总结：总结相似之处（高人才密度、高考核标准、高淘汰率、信息差）
- 人设强化："我是夏树，前大厂产品专家，真诚分享"
- 持续更新：说明会持续更新相关内容

**适用主题**：
- 职场分析类（用比喻解释现象）
- 游戏化表达（让严肃话题更有趣）
- 深度观察类（透过现象看本质）

**示例文章**：my-20

### signal-list-blessing：信号列举-祝福型

**判断标准**：
- 标题包含"晋升信号"、"好运"、"预示"等积极祝福类词汇
- 开头有文学感（伏笔、预兆、时间沉淀等）
- 主体碎片化列举信号/特征（不用"1.2.3."编号）
- 不给方法论，只给信号识别和确认感
- 结尾有"接好运"引导

**典型特征**：
- 开头：文学感悬念（"早已埋下伏笔"、"无声地告诉你答案"）
- 主体：多维度观察（领导、同事、HR、个人状态）
- 碎片化：换行+emoji自然列举，不标明分类标签
- 观察者视角：不说"你应该"，说"你会发现"
- 情感升华：过渡段（"那些...的...，都在为...铺路"）
- 结尾：确认 + 祝福 + 接好运引导（"评论区接好运～"）

**适用主题**：
- 祝福类（晋升、加薪、好事发生）
- 积极心理暗示类
- 不需要数据反馈、只需要情感互动

**示例文章**：others-09（参考文章《晋升信号已出现》）

**核心目标**：激发读者评论"接好运"，通过希望感、确认感、仪式感引发互动

### truth-reveal-perspective：真相揭示-视角递进型

**判断标准**：
- 标题包含反常识观点（"停止X"、"不是X而是Y"）
- 开头用对比或悖论颠覆认知
- 主体用多个视角层层递进（个人→关键角色→系统）
- 揭示读者不知道的隐秘规则/机制
- 结尾有价值升华（上升到人生/价值层面）

**典型特征**：
- 开头：误以为vs真相、不是X而是Y（颠覆认知）
- 多视角递进：你看到的→Leader看到的→公司看到的
- 揭示隐秘规则：读者不知道的机制、考量、逻辑
- 引用金句："和XX聊过，他说得很直接"
- 信息密度极高：每句话都有增量，无废话
- 价值升华：不是祝你好运，而是重新定义价值

**适用主题**：
- 揭示隐秘规则类（绩效、晋升、面试、职场潜规则）
- 颠覆认知类（不是X，而是Y）
- 多视角分析类（从Leader/公司/系统维度分析）
- 有信息差的主题（读者不知道的规则）

**何时使用**：
- 主题有"读者不知道的规则"
- 能从多个视角（Leader/公司/系统）分析
- 有颠覆性观点
- 目标是给真相，不是教技巧

**示例文章**：my-21

**核心成功因素**：
- ⭐⭐⭐⭐⭐ 颠覆性观点：完全颠覆常识
- ⭐⭐⭐⭐⭐ 多视角递进：Leader/公司视角给信息差
- ⭐⭐⭐⭐ 情绪共鸣：强烈反差戳中痛点
- ⭐⭐⭐⭐ 引用金句：增强真实感
- ⭐⭐⭐⭐ 价值升华：提升到人生层面
- ⭐⭐⭐⭐⭐ 信息密度：每句话都有增量

### experience-list：感悟列举型

**判断标准**：
- 开头用个人经历/故事/朋友留言引入话题
- 主体用编号（01、02、03...）列举多个感悟/收获/经验
- 每个感悟有小标题 + 深度展开
- 结尾有建议、鼓励或价值升华

**典型特征**：
- 开头：个人故事/朋友经历/情感表达，引出话题（不是问题痛点）
- 主体：编号列举感悟（通常4-7个），每个感悟有：
  - 小标题（概括核心观点）
  - 具体案例/朋友经历/理论支撑
  - 深度思考和价值提炼
- 真实性：用具体人物故事增强真实感（"一位在广核的朋友"、"华科毕业的山东老乡"）
- 深度：不是简单列举，而是有深度的思考和人生感悟
- 结尾：建议/鼓励/价值升华（不是简单的互动引导）

**与 problem-solution-list 的区别**：
- problem-solution-list：提出问题→给出解决方案（方法论导向）
- experience-list：引入话题→分享感悟（思考/反思导向）

**适用主题**：
- 职场感悟类（离职思考、工作反思、职业成长）
- 经验总结类（几年工作的收获、某段经历的成长）
- 人生思考类（价值观、人生观的总结和升华）
- 成长反思类（对过去的回顾、对未来的展望）

**示例文章**：others-01（参考文章《从核电离职，你可以带走什么？》）

**核心成功因素**：
- ⭐⭐⭐⭐⭐ 真实故事：用具体人物经历增强可信度
- ⭐⭐⭐⭐⭐ 深度思考：每个感悟都有深度，不是表面列举
- ⭐⭐⭐⭐ 情感共鸣：情感表达真挚，容易引发共鸣
- ⭐⭐⭐⭐ 理论支撑：引用权威理论或概念增强说服力
- ⭐⭐⭐⭐ 价值升华：结尾上升到人生/价值观层面

## 如何判断框架类型

### Step 1: 分析文章结构

1. **开头**：用了什么手法？（提问/故事/数据/观点...）
2. **主体**：有几个要点？怎么展开的？（并列/递进/对比...）
3. **结尾**：怎么收尾？（总结/呼吁/提问...）

### Step 2: 提取逻辑链条

- 核心逻辑是什么？
- 论证方式是什么？
- 递进设计是什么？

### Step 3: 对照分类标准

根据 `classification.md` 中的框架类型定义，找到最匹配的类型

### Step 4: 如果无法归类

- 可能是新的框架类型
- 在 `classification.md` 中添加新类型定义
- 创建新的文件夹：`framework-type-[新类型]`

## 语料差异判断

### 维度1：内容类型
- 干货分享类
- 故事叙述类
- 观点输出类
- 方法教学类
- 清单总结类

### 维度2：语气风格
- 专业严肃
- 轻松幽默
- 亲切友好
- 犀利直接
- 温柔治愈

### 维度3：表达方式
- 口语化
- 书面化
- 混合型

## 分类决策流程

```
新文章
  ↓
分析框架结构
  ↓
对照现有类型
  ↓
匹配成功？→ 放入对应文件夹
  ↓ 否
判断是否为新类型 → 是 → 添加新类型定义
  ↓ 否
选择最接近的类型 → 放入该文件夹，并在备注中说明
```

## 分类示例

### 示例1：问题-解决型文章

**文章内容**：开头提出职场新人常遇到的沟通问题 → 分析原因 → 给出3个解决方法 → 总结

**分类过程**：
1. 结构：问题 → 原因 → 方法 → 总结
2. 逻辑：提出问题-分析问题-解决问题
3. 对照标准：匹配"问题-解决型"
4. 分类：`framework-type-a/`（假设 type-a 是问题-解决型）

### 示例2：清单总结型文章

**文章内容**：直接列出"产品经理必备的5个技能"，每个技能简短说明

**分类过程**：
1. 结构：标题 + 5个要点并列
2. 逻辑：并列关系，无递进
3. 对照标准：匹配"清单总结型"
4. 分类：`framework-type-b/`（假设 type-b 是清单总结型）
