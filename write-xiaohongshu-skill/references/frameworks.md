# 可复用框架库

本文档记录从小红书爆款笔记中提取的可复用写作框架。

## 框架模板

每个框架包含：
- **框架名称**
- **适用场景**：适合写什么类型的笔记
- **结构图**
- **关键技巧**

---

## 问题-解决-列举型框架（Problem-Solution-List）

**适用场景**：适合写干货类、方法分享类、经验总结类笔记

**结构**：
```
标题：[问题/痛点 + 解决方案]
├── 开头：现象/痛点引入 + 铺垫背景
├── 主体：
│   ├── 要点1：[小标题 + 详细说明]
│   ├── 要点2：[小标题 + 详细说明]
│   ├── 要点3：[小标题 + 详细说明]
│   └── 要点4：[小标题 + 详细说明]（可选）
└── 结尾：总结/互动/引导关注
```

**关键技巧**：
- **开头技巧**：用现象描述或痛点引入，让读者产生共鸣
- **数字编号**：使用"1、2、3"或"首先、再次、最后"清晰列举
- **小标题**：每个要点有小标题，便于快速阅读
- **详细说明**：每个要点展开说明，给出具体方法或例子
- **结尾互动**：引导评论、私信或关注，增加互动
- **金句点缀**：在关键位置嵌入金句，增强记忆点

**示例文章**：my-01, my-02, my-03, my-04, my-05, my-06, my-07, my-08, my-09

## 理论-应用型框架（Theory-Application）

**适用场景**：适合写方法论类、思维模型类、技能提升类笔记

**结构**：
```
标题：[反常识/颠覆认知的标题]
├── 开头：问题/痛点引入 + 个人经历共鸣
├── 理论引入：
│   ├── 引用权威理论/模型
│   ├── 用生动例子解释理论
│   └── 说明为什么这个理论重要
├── 实际应用：
│   ├── 步骤1：[具体说明 + 示例]
│   ├── 步骤2：[具体说明 + 示例]
│   ├── 步骤3：[具体说明 + 示例]
│   └── ...（根据理论展开）
└── 结尾：鼓励 + 行动呼吁
```

**关键技巧**：
- **标题技巧**：用反常识标题吸引注意（"不是X，而是Y"）
- **个人经历**：用个人经历拉近距离，产生共鸣
- **理论权威**：引用神经科学、心理学等权威理论增强可信度
- **例子生动**：用电影、故事等生动例子解释抽象理论
- **步骤具体**：每个步骤都有具体说明和示例
- **鼓励结尾**：给读者信心，鼓励行动

**示例文章**：my-10, my-23

## 吐槽列举型框架（Rant-List）

**适用场景**：适合写吐槽发泄类、情绪共鸣类笔记

**结构**：
```
标题：[主题] + 十大/至暗时刻/那些XX们
├── 开头：简短引入/或直接进入列举
├── 主体：
│   ├── 1：[槽点场景描述]
│   ├── 2：[槽点场景描述]
│   ├── ...
│   └── 10：[槽点场景描述]
└── 结尾：总结/反转/互动引导
```

**关键技巧**：
- **数字列举**：使用"1-10"或"Top10"（倒序）清晰列举
- **场景真实**：每个槽点都是具体场景，有细节
- **情绪递进**：槽点越来越严重，情绪越来越强烈
- **无需解决**：只列举问题，不给解决方案
- **吐槽直接**：直白表达不满和情绪
- **emoji丰富**：大量使用情绪化emoji
- **互动强烈**：结尾引导读者吐槽共鸣
- **真实性强调**：常说"全部根据真实事件而写"

**示例文章**：my-11, my-12, my-13, my-14, my-15, my-16, my-17, my-18, my-24

## 个人叙事型框架（Personal-Narrative）

**适用场景**：适合写个人成长、人生转折、反思感悟类笔记

**结构**：
```
标题：[时间节点] + [身份标签] + [核心事件]
├── 开头：背景介绍（出生、成长环境）
├── 成长线：
│   ├── 学业/职业发展（顺叙）
│   ├── 高光时刻（外在成就）
│   └── 转折点（但只有我自己知道...）
├── 自我剖析：
│   ├── 性格特质（ISTJ/MBTI等）
│   ├── 内在冲突（理想vs现实）
│   └── 觉醒反思（追问自己）
├── 行动：重大决定（裸辞/转型/开始gap）
├── 未来规划：具体的3件事
└── 结尾：互动引导
```

**关键技巧**：
- **时间线叙事**：从出生到现在，顺叙成长历程
- **反差对比**：外人看vs真实感受
- **自我剖析**：性格、价值观、冲突
- **真诚反思**：不回避脆弱和焦虑
- **具体规划**：未来要做的几件事清晰列出
- **结尾互动**：引导读者思考"你呢？"
- **金句收尾**：提炼人生感悟

**示例文章**：my-19, gap-illusion-work, myself-resignation-8-months-return

## 游戏比喻型框架（Game-Metaphor）

**适用场景**：适合用游戏/比喻分析职场现象的笔记

**结构**：
```
标题：[职场现象] + [游戏比喻] + [情绪emoji]
├── 开头：引入现象 + 核心比喻
├── 主体：用游戏环节作为小标题
│   ├── 🔖环节1：[比喻对应说明]
│   ├── 🙋环节2：[比喻对应说明]
│   ├── 🙈环节3：[比喻对应说明]
│   └── 🙅‍♀️环节4：[比喻对应说明]
├── 分析总结：现象与游戏的相似之处
├── 🤫：专业人设强化
└── 持续更新说明
```

**关键技巧**：
- **核心比喻**：用游戏比喻职场（狼人杀、剧本杀等）
- **emoji分节**：用emoji作为小标题，视觉化强
- **比喻对应**：游戏元素 = 职场元素
- **有叙有议**：既有场景描述，也有深度分析
- **专业分析**：总结相似之处（高人才密度、高考核标准等）
- **人设强化**：结尾强化专业身份
- **持续更新**：说明会持续更新相关内容

**示例文章**：my-20

- **逻辑清晰**：层层递进分析
- **专业性**：行业知识深度
- **客观中立**：平衡多方观点

## 观点-论证型框架（Viewpoint-Argument）

**适用场景**：适合写观点输出、方法论分享、反常识观点类笔记

**结构**：
```
标题：[核心观点] + [修饰/疑问]
├── 开头：观点直接抛出（或反常识引入）
├── 论证/解释：
│   ├── 理由分析（为什么这个观点对）
│   ├── 机制解释（背后的原理）
│   ├── 对比分析（和其他做法的区别）
│   └── 个人经历/案例佐证
├── 建议：具体做法
└── 结尾：金句/呼吁
```

**关键技巧**：
- **观点鲜明**：标题或开头就亮出核心观点
- **论证充分**：用理由、案例、逻辑支撑观点
- **机制清晰**：解释"为什么"和"怎么做"
- **具体建议**：给出可操作的方法
- **金句收尾**：强化观点记忆点

## 信号列举-祝福型框架（Signal-List-Blessing）

**适用场景**：适合写鼓励祝福类、好运接福类、积极心理暗示类笔记

**结构**：
```
标题：[主题] + [情感修饰] + [✨/🎉等积极emoji]
├── 开头：一句话点题 + 制造悬念感
├── 主体：碎片化列举多个信号/特征
│   ├── 维度1的信号（换行分隔，不用编号）
│   ├── 维度2的信号
│   ├── 维度3的信号
│   └── 个人状态/感受的信号
├── 过渡段：情感升华（用"那些...都在..."句式）
└── 结尾：确认 + 鼓励 + 祝福 + 接好运引导
```

**关键技巧**：
- **碎片化列举**：不用"1.2.3."编号，用换行+emoji自然列举
- **多维度观察**：从领导、同事、HR、个人状态等多个角度
- **无明确分类**：不标明"维度一/二/三"等分类标签
- **观察者视角**：像在描述已发生的事实，不是教人怎么做
- **确认感强**：用"不是...而是..."、"不再是...而是..."等肯定表达
- **情感递进**：客观描述 → 主观感受 → 情感升华 → 祝福引导
- **"接好运"引导**：结尾用"接住...已经在路上"等话术+积极emoji
- **情绪价值优先**：不给方法论，只给信号识别和确认感

**情感升华句式模板**：
- "那些深夜改的方案、咬牙扛的压力、主动补的技能，都在为晋升铺路🌱"
- "那些...的...，都在为你...铺路/助力/加速"

**结尾引导模板**：
- "接住这些...，沉淀自己，静待官宣。属于你的...，已经在路上🎉"
- "别忽略这些信号！不是你运气好，是能力早已匹配...不用急着追问结果，稳住节奏，做好当下的每一件事"

**示例文章**：my-22, 参考《晋升信号已出现✨ 职场高光，藏在细节里》（别人的爆款）

**核心目标**：激发读者评论"接好运"，通过希望感、确认感、仪式感引发互动

## 真相揭示-视角递进型框架（Truth-Reveal-Perspective）

**适用场景**：适合写揭示隐秘规则、颠覆认知、多视角分析的职场深度内容

**结构**：
```
标题：[反常识] + [理论概念]
├── 开头：颠覆认知引入
│   ├── 误以为vs真相（对比）
│   └── 核心观点：不是X，而是Y（悖论）
├── 真相1：揭示隐秘机制
│   └── 读者不知道的规则/流程
├── 真相2：重新定义标准
│   └── 读者理解错了的标准
├── 视角1：[关键角色]视角
│   ├── 这个角色的真实考量
│   ├── 具体分析（3-4点）
│   └── 引用金句："和XX聊过，他说得很直接"
├── 视角2：[更高维度]视角
│   └── 更高维度的分析
├── 行动建议：基于洞察
│   ├── 停止/避免（无效行为）
│   └── 具体行动（2-3条）
└── 升华：价值引导
    └── 上升到人生/价值层面
```

**关键技巧**：

**1. 颠覆认知的开头**
- 用对比制造情绪冲击："好学生"vs"扎心的事实"
- 用悖论制造认知冲击："不是对勤奋者的奖励机制"→"存量博弈"
- 核心观点句：每段第一句就是核心观点

**2. 多视角递进**
- 从个人视角 → 关键角色视角（Leader/面试官/HR） → 系统视角（公司/行业）
- 层层深入，像剥洋葱
- 每个视角都给读者"不知道的信息"

**3. 揭示隐秘规则**
- 不是教"怎么做"，而是告诉"真相是什么"
- 信息差：读者看不到、想不到的规则和逻辑
- 具体化：用数字、案例增强可信度

**4. 引用金句增强真实感**
- "和我的+1聊过，他说得很直接："
- 直接引用Leader/HR/面试官的话
- 让读者觉得"这是内行人说的"

**5. 信息密度极高**
- 每句话都有增量，无废话
- 删除任何一句都会影响完整性
- 读者觉得"原来如此，我之前都错了"

**6. 价值升华**
- 不是祝你好运，而是重新定义价值
- "绩效不是我们人生的标准"
- 上升到人生/价值层面

**写作公式**：
```
爆款笔记 =
  颠覆认知的观点
  + 多视角的隐秘规则
  + 引用金句增强真实感
  + 基于洞察的行动建议
  + 价值升华
```

**标题设计**：
- 格式：[反常识] + [理论概念]
- 示例：
  - "大厂绩效：停止无效努力，玩懂存量博弈"
  - "晋升答辩：不是评出来的，而是提前一年埋下结果了"
  - "跨部门沟通：不是靠真诚，而是靠利益绑定"
- 关键词：停止、不是X而是Y、真相、博弈、底层逻辑

**何时使用这个框架**：
- ✅ 主题有"读者不知道的规则"
- ✅ 能从多个视角（Leader/公司/系统）分析
- ✅ 有颠覆性观点
- ✅ 目标是给真相，不是教技巧
- ❌ 纯技巧类（"怎么写简历"）
- ❌ 纯情绪类（"吐槽十大"）

**核心成功因素**：
- ⭐⭐⭐⭐⭐ 颠覆性观点：完全颠覆常识
- ⭐⭐⭐⭐⭐ 多视角递进：Leader/公司视角给信息差
- ⭐⭐⭐⭐ 情绪共鸣：强烈反差戳中痛点
- ⭐⭐⭐⭐ 引用金句：增强真实感
- ⭐⭐⭐⭐ 价值升华：提升到人生层面
- ⭐⭐⭐⭐⭐ 信息密度：每句话都有增量

**与其他框架的区别**：

| 维度 | 真相揭示-视角递进型 | 问题-解决-列举型 | 观点-论证型 |
|------|-------------------|----------------|-----------|
| 目标 | 揭示真相 | 教方法 | 论证观点 |
| 信息 | 读者不知道的规则 | 读者已知的方法 | 作者的观点 |
| 视角 | 多维递进 | 单一（怎么做） | 单一（为什么对） |
| 情绪 | 颠覆认知 | 中性 | 理性 |
| 结尾 | 价值升华 | 总结/互动 | 金句/呼吁 |

**示例文章**：my-21（大厂绩效：停止无效努力，玩懂存量博弈）

**实战案例**：

*案例1：写"绩效沟通SOP"*
```
标题：绩效1on1：不是被动接受，而是资源争夺战

开头：误以为绩效沟通是听取结果，真相是战略博弈

真相1：绩效结果早已在博弈中成型
- +1早在写自评前就心里有数
- 1on1是最后一次对齐和争取机会

视角1：Leader的真实考量
- 这不是单纯的反馈，而是资源分配
- 高绩效的人想要更多资源
- 中等绩效的人要明确方向
- 低绩效的人要准备改进计划
- 引用："和我的+1聊过，他说1on1是分活的关键节点"

视角2：你的博弈策略
- 预判绩效水平
- 系统整理成果，唤醒记忆
- 现场引导：从听到说，争取资源

行动建议：
停止被动接受，学会主动引导

升华：绩效沟通不是判决书，而是翻盘或乘势而上的关键战役
```

*案例2：写"晋升答辩"*
```
标题：大厂晋升：不是评出来的，而是提前一年埋下结果了

开头：误以为答辩决定晋升，真相是早已定型

真相1：晋升在答辩前就成型了
- 半年前+1就开始铺垫
- 评委早就通过汇报了解你

视角1：评委的真实考量
- 不是看答辩表现，而是看是否早已胜任下一职级
- 答辩只是走流程，确认结果
- 引用："和我的+2聊过，他说晋升答辩就是个过场"

视角2：你的应对策略
- 功夫在平日，提前一年布局
- 答辩前+1已经帮你铺路
- 答辩时不被问住，应变能力

行动建议：
停止临阵磨枪，功夫在平日

升华：晋升不是终点，而是持续成长的里程碑
```

## 感悟列举型框架（Experience-List）

**适用场景**：适合写职场感悟类、经验总结类、人生思考类、成长反思类笔记

**结构**：
```
标题：[话题 + 带走/收获/感悟]
├── 开头：
│   ├── 故事/留言/经历引入（具体人物、具体场景）
│   ├── 情感表达（真实的感受、回忆）
│   └── 引出话题（用提问或过渡）
├── 主体：
│   ├── 感悟01：[小标题 + 具体案例/朋友经历 + 深度思考]
│   ├── 感悟02：[小标题 + 具体案例/朋友经历 + 深度思考]
│   ├── 感悟03：[小标题 + 具体案例/朋友经历 + 深度思考]
│   ├── 感悟04：[小标题 + 具体案例/朋友经历 + 深度思考]
│   ├── 感悟05：[小标题 + 具体案例/朋友经历 + 深度思考]（可选）
│   └── 感悟06：[小标题 + 具体案例/朋友经历 + 深度思考]（可选）
└── 结尾：
    ├── 建议：给具体可行的建议（1-3条）
    ├── 鼓励：给读者信心和力量
    └── 升华：上升到人生/价值观层面
```

**关键技巧**：
- **开头技巧**：用具体人物故事/留言引入，不是问题痛点，而是话题引发
- **编号方式**：使用"01、02、03"而不是"1、2、3"，显得更有层次感
- **小标题设计**：每个感悟有两部分——核心观点 + 意象/隐喻（如"冗余的哲学——让自己永远有的选"）
- **真实故事**：用具体人物增强真实感（"一位在广核的朋友"、"华科毕业的山东老乡"）
- **理论支撑**：引用权威理论或概念增强说服力（如"冗余设计"、"质疑的工作态度"）
- **深度思考**：每个感悟不是简单列举，而是有深度的价值提炼
- **情感共鸣**：情感表达真挚，容易引发共鸣（"堆芯余热"、"烙印"）
- **结尾升华**：上升到人生/价值观层面，不是简单的互动引导

**与问题-解决型的区别**：
- 问题-解决型：提出问题→给出解决方案（方法论导向，教你怎么做）
- 感悟列举型：引入话题→分享感悟（思考/反思导向，告诉你为什么）

**示例文章**：others-10（参考文章《从核电离职，你可以带走什么？》）
